What does HTML stand for?

HyperText Markup Language.

From its nascence to today...

  • Early 90s
    • Invented by Tim Berners-Lee
    • Created "hypertext" to share scientific papers
    • First web page August 6, 1991
    • No layout, No styling.
    • A great example! http://www.w3.org/People/Raggett/book4/ch02.html

From its nascence to today...

  • Late 90s
    • HTML 4 in 1997-1998
      • Standardized by w3 Consortium (pack of super nerds)
      • This was a year after CSS (Cascading Style Sheets) was introduced.
      • Plenty of table-based layouts around!

From its nascence to today...

  • Early 00s
    • XHTML in 2000
      • Combined XML and HTML
      • Introduced stricter syntax
    • HTML 5 in 2008-2009
      • Adopted over XHTML 2.0
      • Still HTML, but with many new features

Who uses HTML?

  • Web designers:
    • Plan, design and create (usually) small-scale websites.

  • Web developers:
    • Involved in programming dynamic web applications or large sites.
      • Front-end web developers:
        • Responsible for the user experience on large sites (client-side).
      • Back-end web developers:
        • Responsible for handling and storing data and server file structure (server-side),
        • (But these folks don't really use HTML).

HTML and CSS

HTML:

  • Came first (about 7 years before CSS)
  • Provides logical structure to the content (paragraphs, headings, tables, etc.)
  • Browser's default settings provides rudimentary "style" (spacing, font size, etc.)
  • Before CSS, some additional "styles" were applied with HTML. This is now considered very bad coding!

HTML and CSS

CSS:

  • CSS stands for Cascading Style Sheets
  • Took a while to get websites and browsers to adopt
  • The purpose is to remove all styles from HTML and keep them separate
  • Very flexible and powerful style sheet language that allows websites to look the way they do today
  • Describes how you want your site look — presentation-wise

Elements and Tags

Every HTML element is defined by a tag.

Syntax is important!

<p>

Content goes here.

</p>

HTML is case-insensitive but the accepted convention is to use lower-case (except for the doctype element).

Container Elements

If the element contains content (called a container element), it includes an opening and a closing tag.

Commonly used container elements:

  • <p> (paragraph)
  • <h1> (heading levels 1 - 6)
  • <table> (table)
  • <ul> (unordered list)
  • <ol> (ordered list)
  • <li> (list item)
  • <div> (division)
  • <a> (link)

Go to http://www.w3schools.com/tags/default.asp for a full list of elements

Empty Elements

If the element does not contain content, it is said to be an empty element

Three ways to write empty elements:

  • <br></br>
  • <br />
  • <br>

The self-closing tag syntax is required in XHTML. It is still recommended for clarity.

Empty Elements

Commonly used empty elements:

  • <br /> (break tag)
  • <img /> (image tag)
  • <input /> (form input)
  • <button /> (form button)
  • <hr /> (horizontal rule)

Block-level vs. Inline Elements

Block-level:

  • So far we've just talked about block-level elements.
  • Start on a new line.
  • Browsers give them default padding on top and bottom.
  • Examples are paragraphs, headings, lists, list items, tables and forms.

Block-level vs. Inline Elements

Inline:

  • Do not start on a new line.
  • Must be nested inside a block-level element.
  • Examples are images, links, abbreviations, and elements used to format text.

Inline Elements

Elements that format text:

  • <em> (emphasize)
  • <i> (italicize)
  • <strong> (make strong)
  • <b> (make bold)
  • <span> (has no effect by itself)

Links and Images

Both require attributes to tell the browser more information.

Attributes:

  • Attributes "describe" the element (provide additional information).
  • Attributes are followed by values.
    • think ~ person: hair = "brown"
  • Attributes are always placed inside an opening tag, before the right angle bracket.

  • Syntax:

  • <a href = "http://girldevelopit.com">Girl Develop It</a>

Links

Links have three components:

  • tag: <a></a>
  • href attribute: "http://www.girldevelopit.com"
  • alt attribute: "Girl Develop It"

<a href ="http://www.girldevelopit.com" title = "Girl Develop It Homepage">Girl Develop It</a>

The <a> tag surrounds text or images to turn them into links.

Link continued...

Using target="_blank" causes the link to open in a new window/tab

example: <a href="home.html" target="_blank">Link Text</a>

Inserting mailto:some_email_address.com into the href attribute causes the link to open the default mail client

example: <a href="mailto:info@girldevelopit.com">E-mail us!</a>

Good Housekeeping: Managing Files and Folders

Create a top level folder and put your index.html document inside. This is your "site root folder".

Inside, next to index.html, create a new folder called "styles".

Create another folder next to "styles" called "images". Copy your images into the "images" folder.

Now you have images in a folder, but you need to tell the browser where they are!

Relative vs. Absolute paths for links & images

  • Absolute:
    • Refer to a specific location of a file on a server
      src = "http://www.girldevelopit.com/chapters/detroit"
    • Typically used when pointing to a link that is not within your own domain.
    • Easy to use. There's no need for a starting point.
    • Think ~ searching for an address on MapQuest.

Relative vs. Absolute paths for links & images

  • Relative:
    • Refer to a local file in your site root folder
      src = "images/myimage.jpg"
    • Relative paths change depending upon the page the link is on.
    • Describes the location of the file relative to the file you're in.
    • The current file is your "starting point"
    • Think ~ using the "directions" feature on MapQuest.

What kind of path do you type into your address bar?

Providing "directions" using relative path

If the file is in the same folder:
kitty.jpg

If the file is in a folder on the same level you're on:
images/kitty.jpg

If the file is in a folder that's one level above: ../images/kitty.jpg

If the file is in a folder at the top and you are buried several folders deep:
/images/kitty.jpg

Writing Clean Code

1. Nest your tags properly!

Nesting

All elements "nest" inside other elements...except the HTML element! (every thing else nests inside it)

If you consistently indent your code, you will avoid "bad nesting"!

For example:

Your "a" element nests inside your "p" element, which nests inside your "body" element which nests inside your "html" element.

Whichever element OPENS first CLOSES last!

2. Make good use of white space!

Browsers don't care about white space

  <p>Paragraph 2</p>
  <p>Paragraph 3</p>


  <p>Paragraph 1</p> <p>Paragraph 2</p>  <p>Paragraph 3</p>

3. Leave yourself notes!

You can add comments to your code. The browser ignores them, but you (or another coder) can see them.

<!-- Comment goes here -->

Use them to organize your code:

  • <!-- Beginning of header -->
  • <div id="header">Header Content </div>
  • <!-- End of header -->

Or 'comment out' code (to hide it from the browser):

  • <!--
  • <ol>
  • <li>List Item</li>
  • <li>Another List Item</li>
  • </ol>
  • -->
